Soft Keyboard Handling

Hiding the Soft Keyboard


$.someField.blur(); // cross-platform method
if(OS_ANDROID) {
  // android only method
  Ti.UI.Android.hideSoftKeyboard();
} 

if(OS_IOS) {
  Ti.App.addEventListener('keyboardframechanged', function(){
    Ti.API.info('Soft keyboard shown');
  });
}

Prevent Soft Keyboard from Covering Input

  • Put field in a scrolling view (i.e. ScrollView)
  • On Android, set: 
    
$.win.windowSoftInputMode = Ti.UI.Android.SOFT_INPUT_ADJUST_PAN;

UI Components

Dialog


var dialog = Ti.UI.createAlertDialog({
  cancel: 1,
  buttonNames: ['Yes', 'No'],
  message: 'Delete the file?',
  title: 'Delete'
});
dialog.addEventListener('click', function(e)
  {
  if (e.index === e.source.cancel){
    Ti.API.info('Cancel was clicked');
  }
});
dialog.show();

					

<!-- in the View XML -->
<AlertDialog id='alert'/>

/* in the TSS */
"#alert": {
  cancel: 1,
  buttonNames: ['Yes', 'No'],
  message: 'Delete the file?',
  title: 'Delete'
}

// in the controller
$.alert.addEventListener('click', function(e)
  {
  if (e.index === e.source.cancel){
    Ti.API.info('Cancel was clicked');
  }
});
$.alert.show();
You've probably used alert() frequently in your code. The full api is Ti.UI.createAlertDialog() and supports dialog title, multiple buttons, and even embedded views on Android. Shown here are the API and Alloy techniques for creating an alert dialog. Most times, you do this with api code in the controller.

UI: Switch


<!-- in the View XML -->
<Switch id='theSwitch'/>

/* in the TSS */
'#theSwitch[platform=android]': {
  style: Ti.UI.Android.SWITCH_STYLE_CHECKBOX
  /* or Ti.UI.Android.SWITCH_STYLE_TOGGLEBUTTON */
}

// in the controller
$.theSwitch.addEventListener('change', function(e){
   if(e.value === true) {
      // switch is on ...
   }
});
  • Presents two mutually exclusive choices
  • On iOS, user can turn on indicators: | (on) and 0 (off)
  • Can't change 'On' and 'Off' labels on iOS
  • On Android & Mobile Web you can use a checkbox style and change the labels
  • Add change event listener to switch and check value property to determine state

UI: Slider


<!-- in the View XML -->
<Label id='customLabel'/>
<Slider id='theSlider'/>

/* in the TSS */
'#theSlider': {
  min: 0,
  max: 100,
  value: 3,
  thumbImage: 'images/skull.png'
}
A slider consists of a track and a thumb (a circular control that the user can slide) and optional images that convey the meaning of the right and left values. When people drag the thumb along the slider, the value or process is updated continuously and is displayed in the track.

Slider values are not discrete (meaning whole-number). You'll have to listen for the 'blur' event, round the value, and update the slider's value to create a discrete/integer slider.

You can create a single or multi-column picker. Android can render as a spinner as shown, but by default appears as an option dialog. However, in that mode, it supports only a single column.

Picker Events

  • Listen on 'change'
  • e.selectedValue
  • e.row
  • e.value

$.picker.addEventListener('change',function(e){
  Ti.API.info((e.selectedValue).join(','));
  Ti.API.info(e.row.title);
});

Date Picker


<Alloy>
  <Window backgroundColor="white">
    <Picker id="picker" selectionIndicator="true" type="Ti.UI.PICKER_TYPE_DATE">
    </Picker>
  </Window>
</Alloy>

// in controller
$.picker.value = new Date();

$.picker.addEventListener('change',function(e){
  Ti.API.info(e.value.toLocaleString())
});
As of this writing, Alloy doesn't support the date type picker constructed entirely in XML. For now, you'll have to set the current date (value) in code.

Option Dialog


<Alloy>
  <Window backgroundColor="white" layout="vertical" exitOnClose="true">
    <OptionDialog id="dialog" title="Delete File?">
      <Options>
        <Option>Confirm</Option>
        <Option platform="ios">Help</Option>
        <Option>Cancel</Option>
      </Options>
      <!-- The ButtonNames tag sets the Android-only buttonNames property. -->
      <ButtonNames>
        <ButtonName>Confirm</ButtonName>
      </ButtonNames>
    </OptionDialog>
  </Window>
</Alloy>

// in controller
$.dialog.show();
On iOS, the option dialog is a modal that slides up from the bottom of the screen; on Android, it's a modal that's centered within the window. Clicking a button on iOS closes the modal whereas on Android, you have to select the value then click the button.

Styling Input Components

  • Background colors, graphics, gradients
  • Border width, color, radius
  • iOS: backgroundLeftCap, backgroundTopCap

  • Set normal, selected, and disabled background colors
  • Gradients on iOS only
  • Graphics stretch to fill control size, or are cropped if larger than control
  • On iOS, you can enable special stretching mode via backgroundLeftCap / top cap
    • You define an area on the left/top that won't be stretched
    • one pixel in the middle is stretched to fill most of the rest
    • right/bottom cap is calculated as
      total size – left cap – center 1px
